Cours/Web/Projet front/Gestion des dépendances

Pour gérer les dépendances de votre site front (les bibliothèques css ou javascript utilisées), on a jusqu'à présent utilisé des CDN. Mais lorsque l'on veut :

  1. contrôlez totalement ce qui est utilisé pour votre site
  2. utiliser des outils comme scss pour compiler notre css.

Il est conseillé de télécharger directement les bibliothèque chez soit.

Ceci pose cependant des problèmes de gestion des dépendances :

  • on doit pouvoir savoir exactement ce que l'on a installé
  • on doit pouvoir gérer les versions des bibliothèques installées
  • on doit pouvoir uniquement gérer le projet de notre code (avec git par exemple) sans avoir besoin de sauvegarder le code de personnes tierces.

On utilise pour cela un outil de gestion de dépendances. Nous allons utiliser celui de https://nodejs.org/, nommé npm.

Projet https://nodejs.org/

Dans un terminal, placer-vous dans le dossier contenant votre site puis créez un projet node avec la commande :

npm init

Une fois que vous avez répondu aux questions (en appuyant juste sur la touche entrée si vous voulez), vous créez le fichier de configuration package.json qui contient les informations relatives à votre projet. En appuyant tout le temps sur la touche entrée j'ai obtenu le fichier suivant :

{
  "name": "mon_super_site",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC"
}

On peut maintenant installer un package javascript, comme https://tabulator.info/ pour rendre notre table plus jolie :

npm install tabulator-tables

Le fichier package.json a maintenant changé, il contient les trois lignes suivantes en plus :

  "dependencies": {
    "tabulator-tables": "^5.5.2"
  }

Ainsi qu'un fichier package-lock.json qui contient un descriptif détaillé des bibliothèques installées.

Enfin, les bibliothèques téléchargées l'on été dans le dossier node_modules/ lui aussi nouveau. Dans notre cas, seulement trois bibliothèques ont été installées avec une seule commandes, mais cela peut être beaucoup plus.

Pour utiliser tabulator comme dans l'exemple tables html, il faut avoir accès :

  • à la bibliothèque css qui est dans le dossier node_modules/tabulator-tables/dist/css/tabulator.min.css
  • à la bibliothèque javascript qui est dans le dossier node_modules/tabulator-tables/dist/js/tabulator.min.js

Il n'y a pas de règle pour savoir quoi récupérer. Mais normalement, vous devriez trouver ce qu'il faut dans les dossiers js ou css et prenez les fichiers minifiés (moins lisible pour un humains mais plus léger pour traverser le réseau).

Parfois, la bibliothèque vient également avec son code source, vous avez alors deux dossiers, src/ pour les sources et dist pour les fichiers utilisables.

Modifier le site de la partie précédente pour utiliser les modules de node.

L'intérêt principal de passer via l'installeur de package de node (npm) plutôt que de tout télécharger soit même est que le fichier package.json contient toutes les dépendances de votre projet. Pour partager le projet on a pas besoin d'intégrer' le dossier node_modules/ qui peut être gros, il suffit de le reconstruire à partir du fichier package.json en utilisant, dans un terminal placé à la racine du site, la commande :

npm install

Téléchargez l'archive mon_super_site_node.zip qui contient le site sous la forme d'un projet node :

  1. installez les dépendances
  2. créez un serveur statique pour le servir

Si vous n'installez pas les dépendance, le site ne peut fonctionner puisqu'il ne contient pas de dossier node_modules/.

On conserve également parfois le fichier package-lock.json dans la distribution d'un projet node. Vois le lien suivant pour le pourquoi de son existence et son utilisation courante :
